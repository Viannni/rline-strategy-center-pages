## 2. 产品目标

把当前以资料浏览为主的静态原型，升级为可完整演示 R线一天实际工作方式的高保真策略后台。

主演示链路必须端到端跑通：

```text
用户数据进入
-> 字段校验
-> 基础高优分与运营提分潜力计算
-> H层级识别
-> F12触达准入校验
-> 三类角色与文字/电话路由
-> 任务生成、处理和结构化回写
-> 分数、信号、H层级与主责重新计算
```

成功标准：项目组成员可以只操作一名模拟用户，就理解该用户为什么得到当前分数、为什么进入当前层级、为什么派给当前团队，以及任务回写后系统如何改变下一步动作；产研团队还能从每个动作继续下钻，看到它应该落在哪个现有系统、复用什么能力、改什么字段、补什么接口和如何降级。

## 3. 产品边界

### 3.1 本期包含

- 内置覆盖月课、年课、H1-H4/L、风险、营销和交易信号的模拟用户。
- 支持 CSV/JSON 导入、校验、预览、字段映射和错误报告。
- 支持浏览器本地保存、导出当前快照和恢复初始演示数据。
- 支持策略中心、Agent+指导师、学情沟通组、二销团队四种角色视图。
- 支持逐项计分、规则试算、H层级判断、路由追踪、任务回写和重新计算。
- 支持月课 T0-T28、年课 M1-M12 生命周期查看。
- 支持端内活动、中心化提分和奖学金模块的演示入口。
- 支持数据底座和跨部门提需清单下钻。

### 3.2 本期不包含

- 不连接公司真实学习、CRM、订单、支付、营销或工单接口。
- 不实现真实账号登录、组织权限和数据隔离。
- 角色切换只模拟权限和工作视图，不作为安全边界。
- 不把模拟规则发布到真实 CRM、任务系统或营销中心。
- 不储存真实用户数据；导入数据只保存在当前浏览器中。

### 3.3 可行性优先原则

- 原型不是建议公司重新建设一套与现有后台平行的大系统，而是展示现有系统如何组合成R线策略工作台。
- 每一项功能必须标记为“已确认可复用、已有入口待核对、需要改造、必须新增、可降级”之一。
- 已确认能力与建议能力必须在视觉上区分；没有实读到深层字段的模块不得写成“已经支持”。
- MVP优先复用CRM标签、现有任务台、营销中心活动、工单和可视化看板；复杂分群、AI识别和全自动路由可以后置。
- 产研查看任一功能时，都能看到现有系统、后台路径、已有字段、复用方式、改造项、接口/数据依赖、负责人、验收标准和一期降级方案。

## 4. 用户与角色视图

### 4.1 策略中心

- 查看全局用户池、H层级、任务、风险、团队负载和生命周期。
- 查看与试算基础高优分、运营提分潜力、F13/F14独立信号及风险规则。
- 导入/导出数据、查看字段质量、复盘H层级迁移和任务效果。
- 访问所有模块，但规则修改必须先预览，再明确应用到模拟数据。

### 4.2 Agent+指导师

- 不绑定用户。
- 只看快捷服务、模板问题、待转派问题和本人当班队列。
- 可使用模板回复、记录处理结果、升级到学情/售后/二销。
- 不可修改评分和路由规则。

### 4.3 学情沟通组

- 不绑定用户，由系统派单。
- 一级拆为学情干预、学情规划、售后问题处理。
- 每个小组内部再分文字处理和电话处理。
- 只看所属小组任务、用户证据、推荐动作、SLA和结构化回写入口。

### 4.4 续费沟通组（二销）

- 在续费窗口内绑定用户。
- 月课 T22-T28 处理月课转年；年课 M8-M12 处理年转年。
- 查看绑定用户、蜜续/结续池、F13营销意向、F14交易优先级、全部文字和电话沟通记录。
- 风险用户由二销前台接住，学情/售后提供内部支持；风险解除前禁止强销售动作。

## 6. 视觉与交互方向

- 类型：安静、紧凑、工作导向的运营后台，不采用营销型大卡片或大面积装饰。
- 主色：保留 R线绿色作为品牌和选中状态；深灰用于正文；红、橙、蓝只用于风险、预警和信息语义。
- 密度：桌面端优先，表格和筛选器适合反复扫描；卡片只用于单个指标、任务和可重复对象。
- 圆角：控制在 8px 以内。
- 图标：使用统一图标库；纯图标按钮提供 tooltip 和可访问名称。
- 核心视觉锚点：用户详情顶部固定展示流转轨迹。

```text
行为进入 -> 基础计分 -> H层级 -> 触达准入 -> 主责团队 -> 当前任务 -> 回写 -> 下次重算
```

- 点击总控台指标进入已筛选用户池。
- 点击用户进入完整档案；点击任一分数继续下钻到计分项。
- 角色切换后，同一数据集只展示该角色应看到的队列、字段和动作。
- 所有影响模拟数据的操作提供确认、撤销或恢复方式。

所有业务详情右侧固定提供“产研落位”区域，使用统一状态标签：

- 已确认可复用：2026-06-24后台实读已看到对应字段或操作。
- 已有入口待核对：已确认应用入口存在，但深层字段、权限或接口尚未实读确认。
- 需要改造：现有模块可承接，但要补字段、回写、筛选或权限。
- 必须新增：现有系统之间缺少统一计算、路由、回写或快照能力。
- 可降级：MVP可用标签、人工回写、定时导入等方式先跑通。

## 7. 最新评分模型

### 7.1 模型结构

最新版不是旧版“六维总分”，而是：

- 5个基础评分维度：学习健康、课程体验、成果外化、家长互动、用户适配。
- 2个独立优先级信号：F13营销意向、F14交易状态。
- 1个辅助分：运营提分潜力。
- 1个任务准入字段：F12触达准入/限流。
- 1个硬规则：F15风险熔断优先于销售排序和派单。

### 7.2 基础高优分

```text
基础高优分 =
（学习健康得分 + 课程体验得分 + 成果外化得分 + 家长互动得分 + 用户适配得分）
÷ 已参与维度上限和 × 100
- 风险扣分
```

| 维度 | V0上限 | 主要字段 |
|---|---:|---|
| 学习健康 | 30 | F04、F05、F06 |
| 课程体验 | 15 | F07 |
| 成果外化 | 15 | F08、F09 |
| 家长互动 | 15 | F11、F16 |
| 用户适配 | 5 | F03 |

规则要求：

- 分数限制在0-100。
- F07无有效课程评价时，课程体验维度不参与分母，其他维度重新归一化。
- 未到节点的报告、测评和活动字段显示为“不适用”，不显示为行为失败。
- 报告生成不是加分项；打开、有效停留和分享才进入成果外化。
- F13、F14不得进入基础高优分。
- F12触达超限不得作为基础分扣分。
- 投诉、退款、强负向直接进入H4，不再按总分排序。

### 7.3 F13营销意向

- 输出 L1、L2、L3，不生成一个可与所有用户横向比较的营销分。
- 只有 `exposure_eligible_flag=true` 且营销曝光/分流组一致时，才允许组内排序。
- 未处于同一曝光起点时，仅保留访问、问价、预约、券点击等事件，不改变基础分。

### 7.4 F14交易状态

- P0：下单未付、待付款、支付失败。
- P1：领券未用。
- P2：已领券、券将到期等提醒型状态。
- 仅在续费窗口内生成二销实时任务优先级，不进入基础分。

### 7.5 F12触达准入

- 输出“可派、排队、阻断”及原因。
- 按家长维度跨课程、跨团队计算全局触达次数，并保留渠道明细。
- 渠道硬上限和全局频控在派单前即时校验。
- P0允许豁免，但必须填写原因并对主管可见。

### 7.6 运营提分潜力

```text
运营提分潜力 =
学习基础可修复度 × 30%
+ 成果外化缺口 × 30%
+ 家长可达度 × 20%
+ 活动响应度 × 20%
```

- 仅对H2/H3/L且无风险用户计算。
- 实现时先对“无风险且未直接命中H1”的候选人计算运营提分潜力，再生成最终H层级，避免先分层后算提分潜力造成循环依赖。
- 80-100为强提分；65-79为可提分；40-64为弱提分；0-39暂不提分。
- F10只自动计算端内活动；外部直播手工记录标记为 `MANUAL`，只用于复盘，不与端内自动数据混算。

### 7.7 H层级

判断顺序必须固定：先风险，再判断H1，再判断H2的基础分/成果外化强规则，再用运营提分潜力判断H3，最后归入L；F13/F14只在层级生成后叠加任务优先级。

| 层级 | 进入规则 | 主要动作 |
|---|---|---|
| H1 | 基础高优分≥75且无风险 | 续费窗口内二销24小时跟进；非续费窗口准备学情卡 |
| H2 | 基础高优分65-74，或基础分≥60且成果外化标准化分≥70 | 报告解读、规划、讲座/复习内容铺垫 |
| H3 | 基础高优分40-64、运营提分潜力≥65且无风险 | 中心化活动和低成本提分 |
| H4 | 熔断，或风险扣分≥20，或学习健康<35且有负向反馈 | 停止销售强推，进入体验修复 |
| L | 基础分<40、提分潜力<65且无风险 | 产品化低频维护 |

其中“成果外化标准化分”是该维度0-100的标准化结果，页面同时显示其V0原始得分（0-15），避免把阈值70误解成原始分。

## 8. 16个核心字段的后台表达

数据底座页和用户档案必须统一展示以下字段组：

| 编号 | 名称 | 后台角色 |
|---|---|---|
| F01 | 用户统一ID | 底座，不进分 |
| F02 | 课程与生命周期阶段 | 底座，决定窗口和主责 |
| F03 | 课程级别与用户适配 | 基础分 |
| F04 | 有效完课率 | 基础分 |
| F05 | 近7天活跃学习天数 | 基础分 |
| F06 | 连续漏学天数 | 基础分和漏学任务 |
| F07 | 课程评价分 | 基础分；缺失时分母重算 |
| F08 | 测评/挑战结果 | 基础分和提分潜力 |
| F09 | 成长报告行为 | 基础分和报告任务 |
| F10 | 端内活动参与 | 提分潜力 |
| F11 | 家长回复与可达 | 基础分和提分潜力 |
| F12 | 触达准入/限流 | 派单闸门，不进分 |
| F13 | 营销意向 | 独立信号 |
| F14 | 交易状态 | 独立实时优先级 |
| F15 | 风险熔断状态 | 扣分与熔断 |
| F16 | 任务触达结果 | 结构化回写；次日校准基础分，实时更新F13/F14/F15 |

每个字段详情至少展示：业务名、技术字段、来源系统、当前值、观察窗口、刷新周期、可用状态、是否进分、计算/判断规则、最近更新时间、缺失处理和责任部门。

## 9. 主演示闭环

### 9.1 用户进入与校验

- 用户中心内置不少于24名模拟用户，覆盖月课和年课所有关键阶段。
- 导入后先进入预览，不直接覆盖当前数据。
- 校验用户主键、产品类型、生命周期、日期、枚举、数值范围和跨字段冲突。
- 导入结果显示成功数、警告数、失败数，并允许只导入通过项。

### 9.2 自动计分与解释

- 评分引擎先判断规则适用性，再计算V0原始分、维度标准化分和基础高优分。
- 用户详情同时显示“得分项、未命中项、不适用项、缺失项”。
- 每个得分项显示实际值、观察窗口、数据来源、更新时间和命中规则。

### 9.3 层级与路由

```text
F15风险判断
-> F02续费窗口判断
-> H层级与F13/F14任务优先级
-> 问题类型
-> F12触达准入
-> 文字/电话
-> 团队、小组和具体老师
-> SLA
```

基础路由：

- 模板化常规问题 -> Agent+指导师。
- 非续费窗口的漏学、学习滑落 -> 学情干预小组。
- 报告解读、阶段规划、长期路径 -> 学情规划小组。
- 投诉、退款、售后未解决 -> 售后问题处理小组。
- 续费窗口H1/H2或F14 P0/P1 -> 绑定二销。
- H3且提分潜力≥65 -> 中心化运营，必要时进入学情干预。
- H4 -> 售后/学情修复；续费窗口由二销前台接住但禁止强销售。

文字/电话判断：默认文字；风险、复杂学情、多次文字未解决、明确预约电话或需要高密度解释时转电话。每次判断必须显示命中原因，并允许主管在模拟环境中改派。

### 9.4 任务执行与回写

MVP在界面上收敛为三类任务，详细规则作为子类型：

| 任务大类 | 子类型 |
|---|---|
| 临门转化 | F14待付款/支付失败、领券未用、H1高优转化 |
| 成果外化 | H2证据补强、报告解读、H3提分活动 |
| 风险/漏学修复 | 退款投诉、难度/时间问题、连续漏学 |

老师回写字段包括：是否触达、回复状态、学情结论、营销意向、异议类型、风险变化、下一步动作、下次跟进时间、最终结果。

回写后：

- F13/F14/F15相关事件实时更新独立信号或风险。
- F16保存完整结果。
- 基础分受影响字段在“立即预览”中展示变化，正式模拟批次按次日口径生效。
- 页面并排展示调整前后分数、层级、主责、任务和触达准入变化。

## 10. 页面详细设计

### 10.1 总控台

- 顶部显示当前角色、模拟数据版本、最后更新时间、导入和重置入口。
- KPI：H1/H2、H3提分、H4风险、F14实时任务、F12阻断、数据异常。
- 主区：H层级分布、月课/年课阶段、团队队列与SLA、风险与交易提醒、最近流转记录。
- 所有指标可点击下钻到已筛选列表。

### 10.2 用户中心

- 筛选：产品、阶段、H层级、基础分、提分潜力、F13、F14、风险、触达准入、主责团队、任务状态。
- 表格支持排序、组合筛选、列显示和导出。
- 用户档案包含流转轨迹、基础分、独立信号、风险、字段证据、行为时间线、任务和沟通记录。
- 提供“修改模拟行为并重算”入口。

### 10.3 评分中心

- 标签页：基础高优分、运营提分潜力、独立信号、风险与熔断、规则试算。
- 规则表显示维度、指标、分值、互斥组、适用条件、字段、观察窗口和状态。
- 规则编辑只产生本地草稿；应用前必须预览受影响用户数、层级迁移和任务变化。
- 支持恢复线上基准规则。

### 10.4 进线中心

- 显示新进线、待判定、待分配、已分配、阻断/排队。

核心边界：

## 12. 导入、导出与本地保存

### 12.1 导入

- JSON支持完整用户对象和字段组。
- CSV使用扁平字段模板；布尔、日期、枚举和数值均有固定格式。
- 导入流程：选择文件 -> 解析 -> 字段映射 -> 校验预览 -> 确认导入。
- 主键重复默认更新同一用户；用户可在预览阶段改为跳过。
- 单次演示建议不超过5,000名用户，超出时提示改用抽样数据。

### 12.2 导出

- 导出当前用户与计算结果 JSON。
- 导出当前筛选用户 CSV。
- 导出任务、提需和错误报告 CSV。
- 导出规则草稿 JSON，便于下一次会议继续使用。

### 12.3 本地保存

- 使用带版本号的 `localStorage` 保存用户、任务、规则草稿、角色和最近筛选。
- 数据结构升级时先备份旧快照，再迁移或提示恢复初始数据。
- 提供“撤销最近一次模拟变更”和“恢复初始数据”。

## 13. 异常、空状态与数据安全

必须显式处理：

- F01学习、订单、CRM主键无法映射：不进入自动评分，进入数据异常池。
- F02生命周期非法或与产品类型冲突：停止路由并提示修正。
- F07无评价：显示“不参与分母”，不得显示0分。
- F08节点未到：显示“不适用”，不得判定未完成。
- F09仅生成未打开：触发报告提醒但不加分。
- F10外部直播手工回写：标记 `MANUAL`，不与端内活动自动分混算。
- F12命中限流：任务排队或阻断，基础分不变化。
- F13缺少同曝光/分流组：保留事件，不显示跨用户排名。
- F14在非续费窗口：保留事件，不自动生成二销销售任务。
- F15高分用户命中熔断：直接H4，并解释“风险覆盖分数”。
- CSV/JSON格式错误：逐行返回原因，不破坏当前数据。
- 本地存储不可用或超限：允许继续临时演示，并提示立即导出快照。

导入区域明确提示不要上传真实敏感用户数据；站点不向服务器发送导入内容。

## 14. 响应式与可访问性

- 桌面端按1280px及以上设计，支持1366和1440常见工作屏。
- 平板收起侧栏，保留筛选和完整下钻。
- 手机端保留总控、用户查看、任务查看和回写；复杂规则编辑和大批量导入提示使用桌面端。
- 表格在窄屏使用固定首列、横向滚动或切换为紧凑列表，不截断关键状态。
- 所有交互可键盘访问，焦点样式清晰，抽屉/弹窗具备焦点管理和ESC关闭。
- 颜色不是唯一状态表达；风险、层级和准入均同时展示文字或图标。
- 动效遵循 `prefers-reduced-motion`。

## 15. 测试与验收

### 15.1 规则测试

至少覆盖：

1. 用户A基础分91、F14=P1、无风险 -> H1，续费窗口进入二销优先队列。
2. F07为空 -> 课程体验不进分母，不能把用户错误拉低。
3. 基础分高但退款 -> H4，销售任务被冻结。
4. 有F13营销行为但不属于同一曝光组 -> 不产生跨用户营销排名。
5. F14=P0但不在续费窗口 -> 记录事件，不自动派二销销售任务。
6. F12阻断 -> 任务排队/阻断，基础分和H层级不变。
7. H3且提分潜力≥65 -> 进入中心化提分池。
8. 外部直播 `MANUAL` -> 不进入F10自动分。
9. 报告已生成但未打开 -> 不加成果分，只生成报告提醒。
10. F16回写难度强异议 -> F15更新，并在重算时进入风险逻辑。

### 15.2 导入测试

- 正常JSON、正常CSV、带引号和逗号的CSV。
- 重复用户、缺失主键、非法日期、非法枚举、超范围分值。
- 部分成功导入不得覆盖失败行对应的现有用户。

### 15.3 浏览器验收

- 四种角色切换后导航、队列和可操作按钮正确变化。
- 从总控指标到用户、计分项、路由、任务和回写的链路可连续完成。
- 刷新页面后本地状态保留；重置后回到基准数据。
- 桌面、平板和手机无文字重叠、遮挡或不可达控件。
- 控制台无未处理错误；所有导航和抽屉可正常关闭。
- GitHub Pages线上版本与本地验证版本一致。
- 任一核心能力可以下钻到现有系统落点，且“已确认”和“待核对/需新增”不会混淆。
- 系统落位页完整呈现营销中心、CRM、销售运营、教务、工单、外呼、前台卡片、AI中台和可视化看板的复用与降级路径。

## 17. 已确认决策

- 同时服务策略中心和一线角色，以角色切换区分工作视图。
- 数据形态为模拟数据 + CSV/JSON导入 + 浏览器本地保存。
- 主演示链路为用户进入、自动计分、H层级、派单、回写和重新计算。
- 使用模块化静态后台重构，不引入生产接口和真实权限。
- 业务口径以钉钉线上最新版为唯一事实源。
- 基础高优分与F13/F14营销交易信号必须分离展示和计算。
- 正式落地优先复用公司现有后台，原型作为跨系统能力蓝图和产研参考，不把新建独立后台作为P0前提。
