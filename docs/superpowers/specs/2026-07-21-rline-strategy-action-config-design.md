# R线策略动作配置与48期本机验证面板设计规格

## 1. 文档状态

- 状态：设计已由用户确认，待用户复核本规格后进入实施计划。
- 日期：2026-07-21。
- 目标站点：[R线策略中台后台](https://viannni.github.io/rline-strategy-center-pages/)。
- 业务事实源：[R线全周期策略规划方案V1](https://alidocs.dingtalk.com/i/nodes/r1R7q3QmWe7LqyrKiNoZK6zwJxkXOEP2?utm_scene=person_space)。
- 本期选择：B，本机验证版配置面板。

## 2. 本期目标

把现有后台从“策略与字段展示”升级为“可按人群配置策略动作并验证命中结果”的工作台。

核心链路：

```text
用户数据导入
-> 字段校验与匿名化
-> 评分与H层级识别
-> 生命周期节点识别
-> 风险、触达准入与频控判断
-> 匹配RG策略组
-> 生成文字/电话建议动作
-> 分配三类角色团队
-> 结构化回写
-> 对照48期续费结果验证规则有效性
```

成功标准：

- 策略中心可以按月课T0-T28、年课M1-M12配置不同人群的策略动作。
- 导入48期数据后，页面能说明每个匿名用户为什么命中某个策略、为什么被拦截、为什么分给某个团队。
- 产研能直接看到哪些能力是现有功能可复用，哪些字段必须新增，哪些逻辑需要拍板。
- 真实用户数据只保存在本机浏览器，不进入GitHub、不进入公开演示数据、不在页面展示姓名、私聊内容、IP归属或老师归属。

## 3. 产品边界

### 3.1 本期包含

- 新增“策略配置”一级模块。
- 支持月课、年课两套生命周期策略配置。
- 支持RG01-RG08策略组作为规则模板。
- 支持按H层级、风险状态、生命周期阶段、行为字段、营销信号、交易状态、触达准入组合人群。
- 支持展示策略动作的文字形式、电话准入、承接团队、回写字段和观察指标。
- 支持48期数据本机导入、匿名令牌展示、UID本机检索、评分结果验证。
- 支持策略命中结果与最终续费结果对照，用于判断高优识别是否过宽、过窄或误判。

### 3.2 本期不包含

- 不真实发送私聊消息。
- 不真实创建CRM、外呼或工单任务。
- 不连接公司生产接口。
- 不把48期明细写入代码仓库、seed-data、GitHub Pages或任何远端。
- 不展示真实姓名、私聊原文、电话录音、IP归属、老师归属或可外泄身份信息。

## 4. 面板信息架构

新增一级模块：策略配置。

页面分为四个区域：

| 区域 | 作用 |
|---|---|
| 策略总览 | 展示当前策略版本、适用产品、命中人数、可触达人数、风险拦截人数、电话任务数、文字动作数、待补字段数 |
| 节点策略配置 | 按月课T0-T28、年课M1-M12展示每个节点的人群、内容、动作、电话准入、承接团队和回写 |
| 人群规则构建 | 用字段条件组合目标人群与排除人群，支持查看命中用户样例 |
| 48期验证 | 本机导入K2 48期数据，用匿名用户对照评分、策略命中和续费结果 |

与现有模块的关系：

- 生命周期页负责展示节点路径。
- 评分中心负责解释基础分、H层和独立信号。
- 进线中心负责展示角色流转。
- 策略配置页负责把“节点、人群、动作、角色、回写”组合成可执行规则。

## 5. 规则执行顺序

所有策略匹配必须按固定顺序执行：

```text
1. 风险熔断
2. 生命周期节点
3. 学习状态与H层级
4. 是否进入续费窗口和是否绑定二销
5. F12触达准入与频控
6. RG策略组匹配
7. 文字动作或电话任务判断
8. 三类角色团队分配
9. 回写字段与下次观察周期
```

这样保证系统不会只因“分数高”就触发销售动作，也不会在退款、投诉、强负向或H4情况下继续促销。

## 6. 生命周期节点配置

### 6.1 月课节点

月课按T0-T28配置，每个节点至少有以下字段：

| 字段 | 说明 |
|---|---|
| 节点 | T0-T28 |
| 目标人群 | 新购、未定级、未首课、已首课、漏学、挑战中、H1/H2、领券未付等 |
| 排除人群 | 退款、投诉、强负向、H4、已付款、近7日同类已触达等 |
| 内容类型 | 欢迎、定级、首课、补读、周单词PK、成长报告、讲座、月测、权益、未购归因 |
| 私聊动作 | 要发送的卡片或文字模板类型，不包含真实文案发送 |
| 电话准入 | 明确障碍、主动求助、规划需求、强意向、领券未付、支付异议、风险修复等 |
| 主责团队 | 策略中心、Agent+指导师、学情沟通组、续费沟通组 |
| 回写字段 | 定级、首课、PK、月测、报告打开、回复、异议、风险、支付、未购原因 |
| 观察周期 | 实时、次日、近7日、节点后、续费窗口内 |

关键规则：

- T0-T21以学习承接、效果外化和中心化提分为主。
- T22-T28进入月课转年续费窗口，H1/H2且无风险用户可以由二销绑定承接。
- T22前不得把普通学习提醒直接包装成转化催付动作。
- 测评、PK、月测结果不能和同条优惠券催付混发。

### 6.2 年课节点

年课按M1-M12配置，每个节点至少有以下字段：

| 字段 | 说明 |
|---|---|
| 月份 | M1-M12 |
| 固定学习动作 | 月初校准、3次周单词PK、月末月测 |
| 阶段策略 | 习惯巩固、季度报告、修复、期中规划、假期活动、续费预热、年度收口 |
| 续费状态 | M8开始预热，M11-M12收口 |
| 私聊动作 | 报告卡、预约卡、活动卡、路径卡、权益卡、未续归因 |
| 电话准入 | 学习障碍、规划需求、强意向、支付/决策异议、风险修复 |
| 主责团队 | 学情沟通组或续费沟通组，按窗口和问题类型判断 |
| 回写字段 | PK趋势、月测趋势、报告、预约、到场、异议、领券、支付、未续原因 |

关键规则：

- M1-M7以学习、活动和成果外化为主。
- M8-M12进入续费预热和收口，但风险用户仍优先进入RG07。
- 年课每月固定存在测评与效果外化循环，不能只看M8-M12的营销行为。

## 7. RG策略组模板

策略配置页应把文档中的RG01-RG08做成可下钻模板。

| 策略组 | 对象 | 主要动作 | 电话准入 | 禁止动作 |
|---|---|---|---|---|
| RG01 | 新购、未首课、T0-T10、M1-M2 | 欢迎、定级、首课、周计划、成就反馈 | 高潜未定级、未首课、操作障碍 | 用销售代替首课问题解决 |
| RG02 | 漏学、下滑、T3-T20、M4-M5 | 补读、修复、低频回流 | 多次未动、主动求助、强阻碍 | 电话催学，对H4触发销售 |
| RG03 | 测评、PK、月测、报告节点 | 结果趋势、成长报告、复习直播、解读 | 明确解读、规划需求、学习障碍 | 只报成绩，不给下一步 |
| RG04 | 挑战、比赛、活动节点 | 报名、赛程、进度、结营 | 获奖或权益异常解释 | 社群打卡、群通知 |
| RG05 | 讲座、规划、1V1相关 | 讲座、测评解读、规划邀约 | 同意沟通且满足准入 | 假定固定老师持续跟进 |
| RG06 | T17-T28、M8-M12、H1/H2无风险 | 路径、价值、权益、未购问因 | 强意向、领券未付、支付/决策异议 | 未绑定即全量转二销，对H4促销 |
| RG07 | 全周期风险用户 | 风险确认、修复进展、回流 | 退款、投诉、重大服务问题 | 与销售并发，未解除即回转化 |
| RG08 | 全周期治理规则 | 频控、冲突、版本、复盘 | 不直接拨打电话 | 缺少排除条件即上线 |

每个策略组下钻展示：

- 适用节点。
- 必需字段。
- 可替代字段。
- 当前后台承接能力。
- 必须新增的能力。
- 本期模拟数据如何计算。
- 产研验收标准。

## 8. 人群规则构建

规则构建器不做复杂SQL，而用业务可读条件块。

条件分组：

| 分组 | 字段 |
|---|---|
| 用户基础 | F01用户ID、F02课程与阶段、产品类型 |
| 学习健康 | F04完课率、F05活跃天数、F06漏学天数 |
| 成果外化 | F08测评/挑战、F09报告行为 |
| 家长互动 | F11回复与可达、F16任务触达结果 |
| 活动提分 | F10端内活动参与 |
| 转化信号 | F13营销意向、F14交易状态 |
| 风险与准入 | F12触达准入、F15风险熔断 |
| 计算结果 | 基础分、成果外化标准化分、运营提分潜力、H层级 |

每条策略规则应支持：

- 目标条件。
- 排除条件。
- 动作优先级P0/P1/P2。
- 渠道建议：文字或电话。
- 承接团队。
- 观察窗口。
- 刷新周期。
- 回写字段。
- 生效状态：草稿、模拟中、建议上线、停用。

## 9. 三类角色分配

### 9.1 Agent+指导师

- 不绑定用户。
- 承接快捷服务和模板问题。
- 与Agent可覆盖场景和解决率正相关。
- 适合处理：操作问题、入口解释、模板答疑、简单售后分流。
- 不处理：复杂学情规划、风险修复、续费窗口内转化沟通。

### 9.2 学情沟通组

- 不绑定用户，由系统派单。
- 一级拆为学情干预小组、学情规划小组、售后问题处理小组。
- 每个小组内部分文字处理和电话处理。
- 文字任务适合：报告解读入口、补读方案、活动解释、低压规划确认。
- 电话任务适合：明确学习障碍、连续下滑、多次无改善、主动要求规划、售后风险修复。

### 9.3 续费沟通组（二销）

- 续费窗口内绑定用户。
- 月课转年：T22-T28。
- 年转年：M8-M12，重点M11-M12。
- 负责绑定用户的全部文字和电话续费沟通。
- 学习和售后问题仍可派内部支持任务给学情/售后，但外部用户沟通主线由绑定二销承接。

## 10. 48期本机验证

### 10.1 数据导入原则

- 只导入K2 48期，不混入47期。
- 数据仅保存在当前浏览器localStorage。
- 不写入seed-data.js。
- 不提交到Git。
- 不发布到GitHub Pages。
- 页面默认只展示匿名令牌，例如K2-48-U0187。
- 支持本机UID搜索，用于用户自行核对，不公开显示可识别身份。

### 10.2 导入后字段

48期导入文件建议包含：

| 字段 | 用途 |
|---|---|
| uid | 本机检索和匿名映射 |
| product_type | 验证映射为annual |
| stage_code | 验证映射为M12 |
| renewed_flag | 最终是否续费，用于验证结果 |
| completion_rate | 映射F04 |
| active_days_7d | 映射F05 |
| missed_days | 映射F06 |
| assessment_score | 映射F08 |
| report_opened | 映射F09 |
| activity_participated | 映射F10 |
| parent_replied | 映射F11 |
| touch_status | 映射F12 |
| marketing_events | 映射F13 |
| transaction_status | 映射F14 |
| risk_status | 映射F15 |
| followup_result | 映射F16 |

如果48期实际字段不足，页面必须标出：

- 缺失字段。
- 使用的替代字段。
- 替代字段的可信度。
- 该缺口对H层级和策略命中的影响。

### 10.3 验证指标

验证页至少展示：

| 指标 | 说明 |
|---|---|
| H1命中续费率 | H1用户中实际续费比例 |
| H2命中续费率 | H2用户中实际续费比例 |
| H3提分池续费率 | 被判为可提分用户的实际续费比例 |
| H4拦截准确性 | 风险拦截用户是否确实低转化或高售后 |
| 高优召回 | 实际续费用户中被H1/H2覆盖的比例 |
| 高优精度 | H1/H2用户中实际续费比例 |
| 误判样例 | 高分未续、低分续费、风险解除后续费等 |
| 字段缺口影响 | 缺少某字段时H层变化和策略命中变化 |

### 10.4 用户详情验证

```text
H层级

承接团队
触达准入状态
命中原因

```

## 11. 产研需求表达

每一个配置字段和策略动作都应有产研落位信息：

| 字段 | 说明 |
|---|---|
| 当前是否已有 | 已确认可复用、已有入口待核对、需要改造、必须新增、可降级 |
| 现有系统 | CRM、学习后台、营销中心、订单支付、外呼、工单、数据平台等 |
| 现有路径 | 若已知则展示后台路径；未知则写“需产研核对” |
| 当前字段 | CSI编号、字段名或现有后台名称 |
| R线名称 | R线业务侧想展示的字段名 |
| 取数标准 | 观察窗口、刷新周期、口径 |
| 计算规则 | 加分、扣分、准入、拦截或路由逻辑 |
| 可替代方案 | MVP先用什么字段或人工导入替代 |
| 新增原因 | 为什么现有字段不能满足 |
| 验收标准 | 产研完成后如何判断可用 |

## 12. 关键交互

- 点击节点：打开该节点策略详情。
- 点击策略组：查看适用人群、动作、禁用条件和字段依赖。
- 点击命中人数：进入已筛选用户列表。
- 点击匿名用户：打开评分、策略、任务和续费验证详情。
- 点击“模拟策略”：仅在浏览器中重新计算命中结果。
- 点击“恢复演示数据”：清空本机导入数据，回到公开模拟数据。
- 点击“导出验证结果”：导出不含姓名、私聊、IP归属的匿名CSV或JSON。

## 13. 错误处理和安全提示

- 导入文件含姓名、手机号、私聊文本、IP归属等字段时，默认不展示并提示用户本地敏感字段已隔离。
- 若CSV缺少uid，无法建立匿名映射，导入失败。
- 若stage_code与product_type冲突，进入数据异常池，不参与评分。
- 若F15风险字段缺失，不自动判定为无风险，而是显示“风险字段缺失，需人工核对”。
- 若F12触达准入缺失，策略可以命中，但任务状态显示为“待准入校验”。
- 若最终续费结果缺失，该用户可参与策略命中演示，但不参与精度/召回验证。

## 14. 实施约束

- 所有真实48期明细只能通过前端导入。
- 公共seed数据保持模拟，不混入真实UID。
- 新增代码必须沿用现有静态站结构、localStorage状态和ES module模式。
- 评分与路由逻辑尽量复用现有core/scoring-engine.js与core/routing-engine.js。
- 策略配置应优先用数据配置驱动，避免把T0-T28和M1-M12逻辑散写在视图中。
- 测试需覆盖规则匹配、风险优先、续费窗口、匿名导入、字段缺失、导出脱敏。

## 15. 待拍板问题

- H1/H2/H3/H4/L的最终阈值是否按现有方案先做V0，还是需要48期数据跑完后校准。
- F12电话频次上限和P0豁免规则是否已有公司统一口径。
- M8-M12中“绑定二销”的具体触发边界，是按阶段默认绑定，还是按H1/H2和明确意向绑定。
- 48期可提供的逐用户字段是否足够覆盖F04-F16，若不足需要明确替代字段。
- 导出验证结果是否允许包含本机匿名ID与UID映射，建议默认不导出映射表。
