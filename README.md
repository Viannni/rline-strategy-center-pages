# 英语业务策略经营中台

这是一个可公开分享的静态原型，用于向产研和项目组说明策略团队后台应该如何管理英语各业务线、各级别和各班期的策略资产、用户圈选、下发追踪、效果复盘和数据提需。

R线是首发样板线；K线、E线以同一套数据模型保留结构接入能力。

在线原型：<https://viannni.github.io/rline-strategy-center-pages/>

规则事实源：[R线用户分数系统与高优用户识别方案（最新版）](https://alidocs.dingtalk.com/i/nodes/dQPGYqjpJYgoM27OcBp667plWakx1Z5N?utm_scene=person_space)

## 使用边界

- 所有页面、数据和交互均为本地静态模拟，不连接生产系统或真实业务人员。
- 策略资产统一记录业务域、策略范围、复用标记、差异配置、生命周期节点、负责人、状态、数据依赖和观察窗口。
- 原型不展示真实姓名、私聊内容、通话录音、IP 归属、教师归属或 K2 48 期真实用户级明细。
- 请勿向本地导入真实个人信息、订单号、沟通记录或任何未获授权的数据。

## 本地运行

要求 Node.js 20+ 与 Python 3：

```bash
npm test
npm run serve
```

打开 <http://localhost:4173/>。页面使用版本化 `localStorage` 保存本地模拟操作；浏览器存储不可用时会退化为内存模式。

## 目录说明

- `core/`：策略领域、选择器、导入导出和本地状态存储。
- `data/`：R/K/E 业务域、生命周期和策略资产的模拟数据。
- `views/`：英语全线策略经营视图。
- `tests/`：策略领域、安全边界、交互和 UI 契约测试。
- `output/playwright/`：桌面端与移动端验收截图。

GitHub Pages 从仓库 `main` 分支根目录发布，无服务端依赖。
